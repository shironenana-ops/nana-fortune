# MOSH＋fincode 運用手順

## 現在の方式

公式公開情報で白音七向けAPI・Webhookの具体的利用仕様を確認できていないため、初期運用は手動照合です。

1. 利用者が白音七へ登録・ログイン
2. 同じメールアドレスでMOSHへ申込み
3. 運営者がMOSHの申込み・契約・支払い状態を確認
4. 白音七ユーザーとメールアドレスを照合
5. 照合できた場合だけ `light` / `premium` / 音声1回分を反映
6. 日時、商品、MOSH側識別情報、白音七ユーザー、実施者を秘密情報を除いて記録

自己申告だけで付与しません。メール不一致時は付与せず、本人へMOSH登録情報の確認を依頼します。安全な管理者更新機能は未実装なので、ランク・音声枠の実更新は次PRで設計します。

## 商品別

- ライト：既存内部値 `light`。無料＋ライト鑑定を新規作成可能
- プレミアム：既存内部値 `premium`。無料＋ライト＋深掘り鑑定、会員向け音声利用権
- 音声単体：既存コードでは `extra_voice_remaining` が月次残数の後に消費される。購入確認後の `+1` が自然だが、二重付与防止台帳と安全な管理方法がないため今回自動付与しない

`monthly_voice_limit - monthly_voice_used` を先に消費し、残数がなければ `extra_voice_remaining` を1減らす実装です。

## 例外運用

- 二重購入：MOSH側識別情報と過去証跡を照合し、同一購入を再付与しない
- ライト→プレミアム／プレミアム→ライト：MOSHでは現在の契約を解約し、希望プランへ申し込む。重複契約期間を確認してから反映
- 解約：MOSHで契約状態を確認後、新規利用権を停止。一度届けた履歴は削除しない
- 支払い失敗：MOSHは数日おきに再決済し、複数回失敗で強制解約の可能性がある。解消確認まで有料権限を新規付与しない
- 返金：MOSH上の記録と利用状況を確認し、メール等で証跡を残して対応。条件を独自に断定しない
- 月次：契約継続、決済失敗、解約、ランク、音声付与台帳を照合

## MOSHへの問い合わせ文案

> 外部会員サイト「白音七」と購入者情報を安全に照合する方法を確認したく、ご教示ください。外部サイト向けAPI／Webhookの提供、購入・解約・支払い失敗通知、契約者一覧CSV、申込みID・契約IDの確認方法、サブスク状態取得、推奨連携方法、テスト決済方法、自分の商品を自分で購入できるか、返金時の手数料、解約反映タイミング、fincode 7％以外の手数料、300円商品の精算額の端数処理について、現行仕様をご案内いただけますでしょうか。

公式公開情報で確認できない項目は、回答を得るまで推測で自動化しません。
